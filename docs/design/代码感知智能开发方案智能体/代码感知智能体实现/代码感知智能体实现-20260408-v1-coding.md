## 1. 核心设计原则

**Tool = 眼睛（机械提取），Agent = 大脑（LLM 理解）。** Tool 绝不做判断和总结。

---

## 2. 角色定位

- **角色**：CodeAwareness -- 代码感知专家
- **输入**：projectPath（来自用户请求）
- **职责**：调用 4 个提取器 + 1 个索引器，综合生成 ProjectProfile + ArchTopology
- **输出**：ProjectProfile（7 维度 JSON）+ ArchTopology（分层拓扑 JSON）+ IndexStatus

---

## 3. 执行流程（ReAct 循环）

```
ScanNode.execute(state)
  -> agentRouter.route(CODE_AWARENESS, state)
    -> 5 步 Tool 调用 + 1 步 LLM 综合：
       1. ProjectScanTool(projectPath)           -> 目录树、模块列表、文件统计
       2. DependencyAnalysisTool(projectPath)     -> pom.xml 依赖清单、版本号
       3. CodeStructureAnalysisTool(projectPath)  -> 注解扫描、层依赖、违规检测
       4. ConfigScanTool(projectPath)             -> 配置项提取（脱敏）
       5. CodeIndexTool(projectPath)              -> 向量索引（可跳过）
       6. LLM 综合推理                            -> ProjectProfile + ArchTopology JSON
```

## 4. 工具集

| Tool | 用途 | 调用时机 |
|------|------|----------|
| devplan_project_scan | 目录结构+模块+统计 | 第 1 步，必调 |
| devplan_dependency_analysis | pom.xml->依赖清单 | 第 2 步，必调 |
| devplan_code_structure | 注解扫描+层依赖 | 第 3 步，必调 |
| devplan_config_scan | 配置提取+脱敏 | 第 4 步，必调 |
| devplan_code_index | Qdrant 向量索引 | 第 5 步，可选 |

---

## 5. ProjectProfile 7 维度

| # | 维度 | 数据来源（Tool） | 理解方式（Agent/LLM） |
|---|------|-----------------|---------------------|
| 1 | 项目概述 | ProjectScanTool + DependencyAnalysisTool | 从模块名、依赖推断项目用途 |
| 2 | 技术栈 | DependencyAnalysisTool | 总结关键框架和版本 |
| 3 | 代码结构 | ProjectScanTool | 标注各目录职责 |
| 4 | 现有 API | CodeStructureAnalysisTool | 归类 REST 端点为业务能力 |
| 5 | 数据模型 | CodeStructureAnalysisTool | 识别核心 Entity 及关系 |
| 6 | 架构规范 | CodeStructureAnalysisTool（import 分析） | 判断分层合规性 |
| 7 | 配置概要 | ConfigScanTool | 识别关键外部依赖 |

## 9. 核心业务规则

- R1：Tool 只做机械提取，Agent 做理解和判断
- R2：必须调用全部 4 个提取器后再做综合分析
- R3：只分析不设计，设计是 SolutionArchitect 的事
- R4：输出严格 JSON，供后续 Node 程序化消费
- R5：CodeIndexTool 可选但推荐，已索引未变更自动跳过
- R6：ProjectProfile 必须覆盖 7 个维度，缺少则标注 unavailable

---

## 10. 异常处理要点

- projectPath 不存在 -> Tool 返回 error JSON，Agent 观察后返回失败信息
- 非 Maven 项目 -> Agent 标注技术栈未知
- Qdrant 不可用 -> indexStatus = DEGRADED，不阻断 Profile 生成
- Agent 输出非法 JSON -> ScanNode 重试 1 次，仍失败标记 FAILED
