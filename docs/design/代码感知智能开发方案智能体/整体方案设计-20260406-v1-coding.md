## 1. 核心业务规则

- 规则1：同一 projectPath 只维护一个 Qdrant collection；重复索引请求先检查 file_hash，未变化则跳过
- 规则2：索引粒度为 Java 类级别，embedding 文本 = 类 Javadoc + 类声明 + 所有 public 方法签名（不含方法体）
- 规则3：架构分层校验：api.controller 只依赖 application，application 可依赖 domain，infrastructure 实现 domain 接口，禁止反向依赖
- 规则4：命名规范：Controller/UseCase/Repository/Impl 后缀强制，PascalCase/camelCase 强制
- 规则5：类设计所有类名必须包含完整包路径（全类名），否则验证不通过
- 规则6：方案验证失败后最多重新生成 3 次，超过则标记问题交由人工
- 规则7：单次方案生成总 Token 不超过 50,000
- 规则8：传感器执行顺序：计算型（ms 级）先于推理型（s 级）

## 4. 数据结构

### 关键表及字段

```sql
-- 代码索引状态表
CREATE TABLE code_index_status (
  id BIGINT AUTO_INCREMENT PRIMARY KEY,
  project_path VARCHAR(500) NOT NULL,
  collection_name VARCHAR(100) NOT NULL,
  doc_count INT NOT NULL DEFAULT 0,
  status VARCHAR(20) NOT NULL COMMENT 'IDLE/INDEXING/READY/FAILED',
  last_indexed_at DATETIME,
  file_hash VARCHAR(64),
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  UNIQUE INDEX uk_project_path (project_path)
);

-- 方案生成记录表
CREATE TABLE dev_plan_record (
  id BIGINT AUTO_INCREMENT PRIMARY KEY,
  project_path VARCHAR(500) NOT NULL,
  requirement TEXT NOT NULL,
  document LONGTEXT,
  impact_analysis JSON,
  validation_score INT,
  validation_issues JSON,
  status VARCHAR(20) NOT NULL COMMENT 'PROCESSING/COMPLETED/FAILED',
  model_used VARCHAR(50),
  token_usage INT,
  elapsed_ms BIGINT,
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  INDEX idx_project_path (project_path),
  INDEX idx_created_at (created_at),
  INDEX idx_status (status)
);
```

### 关键 DTO/Record 字段

```java
// DevPlanRequest
```

## 5. 重要约束与边界

- 并发控制：code_index_status 的 status 字段做 CAS（IDLE→INDEXING），同一项目同时只允许一个索引任务
- 事务范围：方案生成无事务要求，每次独立 INSERT
- 降级策略：向量库不可用时降级为纯 LLM 分析（无 RAG），在结果中标注降级
- 不处理的场景：多项目联合分析、实时代码变更监听、代码自动生成
- Token 预算：单次生成总 Token < 50,000，超预算时截断低优先级章节
- 缓存策略：ProjectStructure 和 ArchTopology 缓存 10 分钟（Caffeine）

## 7. 异常处理要点

- 项目路径不存在 → 抛出 `IllegalArgumentException` / 返回错误码 `40001`
- 无 Java 源文件 → 抛出 `UnsupportedProjectException` / 返回错误码 `40003`
- 索引超时（>5min） → 抛出 `IndexTimeoutException` / 返回错误码 `50002`
- LLM 调用失败 → 重试 2 次，仍失败抛出 `LLMProviderException` / 返回错误码 `50001`
- 向量库不可用 → WARN 日志 + 降级为无 RAG 模式，不抛异常
