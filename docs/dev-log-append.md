### 核心功能

**A/B 对比测试**：
1. **基础配置**
   - 选择现有模板作为基准（A 版本）
   - 选择 LLM 模型
   - 选择输出模式（流式/非流式）
   - 配置共用变量（JSON 格式）

2. **Prompt 内容对比**
   - A 版本：显示原始模板内容（只读）
   - B 版本：可编辑的测试版本，基于 A 版本修改
   - 支持一键重置 B 版本为原始版本

3. **并行执行测试**
   - 同时调用两个版本的 Prompt
   - 使用相同的变量和模型
   - 独立显示执行状态（加载中/成功/失败）

4. **结果对比展示**
   - 左右分屏显示 A/B 版本输出
   - Markdown 渲染，支持代码高亮
   - 显示 Token 使用量和执行耗时
   - 方便直观对比效果差异

### 使用场景

1. **Prompt 优化**
   - 基于现有模板进行微调
   - 对比优化前后的输出质量
   - 快速验证改进效果

2. **参数调试**
   - 测试不同的提示词表达方式
   - 对比不同的指令顺序
   - 验证变量替换效果

3. **质量评估**
   - 并行对比多个版本
   - 选择最佳输出方案
   - 数据驱动的决策

### 技术实现

**并行执行**：
```javascript
await Promise.all([
  executePrompt('A', promptA.value, variables),
  executePrompt('B', promptB.value, variables)
])
```

**变量渲染**：
```javascript
let renderedPrompt = promptContent
for (const [key, value] of Object.entries(variables)) {
  const placeholder = `{${key}}`
  renderedPrompt = renderedPrompt.replaceAll(placeholder, value)
}
```

**状态管理**：
- 每个版本独立的状态（loading/success/error）
- 独立的内容、Token 使用量、执行时间
- 实时更新，互不干扰

### 核心设计决策

1. **共用变量**
   - 确保对比的公平性
   - 只改变 Prompt 内容，其他条件相同

2. **并行执行**
   - 提高测试效率
   - 减少等待时间
   - 使用 Promise.all 实现

3. **独立状态**
   - 每个版本独立的加载/成功/失败状态
   - 一个失败不影响另一个
   - 清晰的错误隔离

4. **Markdown 渲染**
   - 复用 Chat.vue 的渲染逻辑
   - 统一的样式和代码高亮
   - 提升可读性

5. **非流式优先**
   - 对比测试默认使用非流式
   - 避免流式输出的复杂性
   - 结果更稳定可靠

### 变更原因

用户需要对比不同 Prompt 版本的输出效果，以便优化提示词质量。通过 A/B 对比功能：
1. 快速验证 Prompt 改进效果
2. 数据驱动的优化决策
3. 提高 Prompt 工程效率
4. 降低试错成本

这个功能是 Prompt 工程的重要工具，帮助用户系统化地优化提示词，提升 LLM 输出质量。
